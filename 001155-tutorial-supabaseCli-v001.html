21  17b. Step 5a: Supabase CLI command

The supabase CLI is a command that you can run from your terminal to control and configure your Supabase server. The name of the command is appropriately “supabase”.

While, you can configure and manage your remote supabase server directly from the supabase website, it is often easier to use the supabase CLI command to perform much of the configuration and management of the supabase server.

Do not confuse the supabase CLI command with the supabase server. These are different products. Both can be installed locally.

  • The supabase server can be installed locally or used only remotely. In this tutorial we will NOT install the supabase server locally and only use a remote supabase server (this is much easier for beginners).

  • We WILL install the supabase CLI tool to enable us to configure and manage our remote supabase server by running commands in our terminal on our local computer.

21.1 Installing the Supabase CLI

There is more than one way to install the supabase CLI:

In the previous section you setup a Supabase project on the Supabase website. Supabase has many different features (database, authentication, edge functions, storage buckets, etc). You can configure most of what you need to configure by “clicking around” on the website. However, there are some things that cannot be done on the website. For these things we must use the “supabase CLI tool”.

In addition, using the supabase CLI command can often be easier and more efficient than accessing the same functionality via the Supabase website.

21.2 Using Supabase CLI via npx

There are different options for installing/running the supabase CLI command. The simplest and easiest way to use the supabase command is to use the npx command to run the supabase command. npx (Node Package Executer) comes with Node.js. While the npm (Node Package Manager) tool can be used to download and install packages on your computer, the npx package is used to download and run the code from an npm package without actually “installing” the package on your computer.

For example the if you run the following command from the top level folder of your project, you will see a quite lengthy message that describes the different options you can use with the Supabse CLI tool. We will be covering some of the many options as we progress through this tutorial.

npx supabase --help

21.3 Installing Supabase CLI on your computer

NOTE: remember that the supabase CLI is NOT the same as the supabase server. In this section we are discussing installing the supabase CLI on your local computer - not the supabase server.

In this tutorial we will be using npx to run the supabase CLI. However there are some benefits to acutally “installing” the supabase command on your computer. If you use npx to run the supabase CLI, then every time you run the npx supabase command, the latest copy of the supabase npm package is downloaded to your computer. If you don’t have access to the Internet, things become more complicated. For this and other reasons, some people prefer to install the supabase CLI on their computer. You can do so by using one of the options listed below. Note that if you do install supabase on your computer then the command would simply be

  • supbase ...(other options)...

instead of

  • npx supabase ...(other options)....

See official instructions for how to install here: https://supabase.com/docs/guides/local-development/cli/getting-started The following summarizes your options to install locally:

  • Install the supabase CLI command as an npm package. To do so, run the following command from the root (top level folder) for your project:

    npm install supabase --save-dev

    NOTE: installing using the -g option to npm is no longer supported

  • You can also install the supabase CLI command on Mac by using the Homebrew package manager or on Windows by using the Scoop package manager. See the instructions here https://github.com/supabase/cli?tab=readme-ov-file to learn more about this option.

  • You can also download the actual program files from here: https://github.com/supabase/cli/releases. Be aware that this page lists “pre-release” versions as well as “release” versions. The “pre-release” versions may have the latest fixes and modifications but are not supported. For example, at the time of this writing the latest “release” version is 2.58.5 but there are “pre-release” versions 2.58.6, 2.59.*, 2.60.1, 2.60.2 Use the pre-release versions if you want the absolute latest features but be aware that they may not have been fully tested.

21.4 How to use the supabase command via npm and npx

IMPORTANT: If you are using supabase cli via npm then you should prefix all uses of the supabase command with “npx” (see examples below). For example, the following command will show you which version of the supabase cli you are using:

npx supabase --version

The following gives you a very short summary for how to use the supabase command

npx supabase --help

npx is the Node Package eXecution command. This command can be used to run (or eXecute) a command that is installed as an npm package.

21.4.1 supabase ‘sub-commands’

The supabse command has numerous “sub commands”. For example:

  • npx supabase db is used to work with the Supabase database.
    Run npx supabase db --help to get help specific to the db sub-command.

  • npx supabse functions is used to work with Supabase “edge functions”.
    Run npx supabase functions --help to get help specific to the db sub-command.

  • etc.

The full official reference for how to use the supabase CLI command is here: https://supabase.com/docs/reference/cli/introduction There are many options and the official reference can be overwhelming. We will walk you through an introduction of using the supabase CLI command in this tutorial.

21.5 Login in to Supabase

Run the following command:

npx supabase login

  • You should see the following: Hello from Supabase! Press Enter to open browser and login automatically.

  • Press Enter and a browser will open and display a code. Copy the code.

  • Go back to the terminal and enter the code and press Enter

  • You should now be logged into supabase.

21.7 Find your project’s reference ID

In the next step we will “link” the project on your computer to your Supabase project that you setup on the Supabase website. To do so, you will need to know your “project ref”. It is listed in the output of the npx supabase projects list command (see above). For example, the REFERENCE ID for the 2nd project listed above is bvysrsysihcpsomfkyja.

The project reference ID is also part of your project’s URL. You can find your project URL in your Supabase dashboard URL. It looks something like this: https://app.supabase.com/project/bvysrsysihcpsomfkyja The part after /project/ is your project ref, ie. in the example above this would be bvysrsysihcpsomfkyja. However, your own project-ref will obviously be different.

21.7.1 Link your project

Once you have identified your project ref, open your terminal, and navigate to your project directory. Then run the following command where “YOUR_PROJECT_REF” is replaced with your actual project ref. This command creates a connection between your local machine and your cloud project so that future commands from the CLI will knows which project you’re working with.

npx supabase link --project-ref YOUR_PROJECT_REF

For example, given the example above the command would be: The ouput shown below is what I got when I ran the command. This indicates that my project is linked with the remote supabase server. The WARNING that you see is harmless. Click on the link below to understand more about this warning message.

npx supabase link --project-ref bvysrsysihcpsomfkyja
Initialising login role...
Connecting to remote database...
Finished supabase link.
WARNING: Local database version differs from the linked project.
Update your supabase/config.toml to fix it:
[db]
major_version = 17
click for an explanation of the WARNING message

Developers often run a local version of the Supabase server that allows them to develop their project without having Internet access. When the developer wants to synchronize the work that they did on their local computer with the remote Supabase server, they can link their project to the remote project (as we did here) and then run the command npx supabase push to “push” their work to the remote server. For this to work correctly, the supabase version that is installed locally must be compatible with the supabase version on the remote server.

We decided that for simplicity, we will not use a local installation of the Supabase server. However, when this git repo was originally developed, the developer did have a local Supabase server installed on his computer. The file config.toml that is in the supabase folder stores configuration information for the local Supabase server. When this project was originally developed, the supabase/config.toml file got committed to git.

The warning message is simply saying that the recorded version of the local Supabase server not compatible with the version for remote Supabase server. However, as we said above, we are not using an actual local Supabase server. Therefore we can ignore this message. Alteratively, we can change the contents of the supabase/config.toml file by modifying the lines in the supabase/config.toml that record the local supabase version to read as follows:

[db]
major_version = 17

21.7.2 Test the link

You can run the following commands to test that your link has been established correctly.

The following should show all Supabase projects your CLI account has access to:

npx supabase projects list

The following should show whether your local directory is currently linked to a Supabase project:

npx supabase link status
