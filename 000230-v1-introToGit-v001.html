6  6. alternate chapter - Intro to Git & GitHub (hands-on, Bash-first)

Goal: learn core Git commands by building a tiny website locally, then pushing it to GitHub. Focus: git init, git status, git add, git commit, git log, git remote add, git push, git clone, git pull.

6.1 0) Setup

6.1.1 Install Git (includes Git Bash on Windows)

  • Windows: Download “Git for Windows” from the official Git site and install. Choose Git Bash when prompted. Launch Git Bash after install.
  • macOS: xcode-select --install (then git --version) or install via Homebrew: brew install git.
  • Linux: Use your distro’s package manager, e.g., sudo apt-get install git.

6.1.2 Configure Git (one-time)

git --version
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main

6.2 1) Create a project & local repo

mkdir hello-site   # create a new folder (i.e. "directory") named "hello-site"
cd hello-site      # change the working directory to hello-site
git init           # prepare this folder for use with git
git status         # show the current status of the git repo

Files created so far

hello-site/
└─ .git/        # (hidden Git database)

6.3 2) Create your first file in the git repository

6.4 2a) “text files” vs “binary files”

All files stored on a computer can be categorized as one of the following:

  • Text files contain only “text” (i.e. letters, numbers, special characters such as !@#$%^&*()_+=-;’:“,./<>?) A text cannot contain any pictures, charts, colors, etc.

  • Binary files may contain any of the above types of content.

There are two types of programs for typing text.

  • text editors - these are only able to create text files. They don’t have the ability to create pictures, colors, charts, etc.

  • word processors - these are programs such as Microsoft Word, Pages (on Mac), Excel, or any program that can create anything other than just plain text

Computer programs for just about all programming languages must be stored in text files and must be written using a text editor program.

Computer programs should NOT be written with a word processor. This is because even if a word processor program is used to create a file that just contains text (and no pictures) the file is still stored in a binary file.

Therefore when creating or editing any code files in your project make sure to use a text editor (not a word processor). The following come installed be default

  • Notead (windows)

  • TextEdit (mac) - Unfortunately TextEdit will not work as a “true” text editor unless you setup “plain text mode” in TextEdit. See below for how to do so:

6.4.1 How to put TextEdit (on mac) in plain text mode:

For a document that’s already open:

Go to the Format menu at the top of your screen Click Make Plain Text (or press Shift+Cmd+T)

To make plain text the default for new documents:

  1. Open TextEdit

  2. Go to TextEdit menu > Preferences (or press Cmd+,)

  3. In the New Document tab, select Plain text instead of “Rich text”

  4. While you’re there, you might also want to uncheck “Smart quotes” and “Smart dashes” in the Options section, as these can cause issues with code

Once you do this, all new TextEdit documents will open in plain text mode by default, which is much better for writing code since it won’t add any hidden formatting characters. You’ll notice the difference immediately - in plain text mode, you won’t see font formatting options in the toolbar, and your text will appear in a monospace font (usually Monaco or Menlo), which is ideal for code.

6.5 2b) Add a simple HTML page

Websites are created using many different programming languages and technologies. Technologies used to implement websites range from simple websites that use just HTML, CSS and JavaScript, to very complex websites that incorporate many other technologies. However, almost every website has a collection of the following 3 types of files

  • HTML files (usually end with .html)

    These files contain the text of what is displayed on the website. Note that more complex websites often use techniques that generate a lof of the text for the website with other techniques.

  • CSS files (usually end with .css)

    These files contain the instructions for the “styling” of the website - such as colors and fonts for different sections of the website.

  • JavaScript files (usually end with .js)

    These files contain code that can make a website more interactive and reactive to user input.

We will create our first code file, index.html below. To create the file we can use one of the following two techniques:

6.5.1 Using a text editor to create our first code file

We want to create a file named index.html that contains the HTML code shown below. To do so you can create the file in a text editor such as Notepad (on Windows) or TextEdit ( in “plain text” mode - see above) on Mac. Make sure to save the file in the folder that you created above. To see the name of the folder where you should save the file you can type the Bash command pwd (pwd stands for Print Working Direcotry).

<!doctype html>
<html>
  <head>
    <meta charset="utf-8">
    <title>Hello Site</title>
  </head>
  <body>
    <h1>Hello, Git!</h1>
    <p>This is my first version.</p>
  </body>
</html>

6.5.2 ALTERNATIVE - use a bash command to create the file

Alternatively you can use the long command shown below in Bash directly to create the file.

Things to pay attention to:

  1. The comamnd starts with the line cat << EOF > index.html.
    The name of the file you are creating is specified after the > sign. In this case, the name of the file is index.html.

  2. The command ends with a line simply says EOF (no spaces before or after). The letters EOF stand for “End Of File”. Note that you can replace EOF on the first and last lines with any unique set of

  3. The contents of the file you want to create is placed between the first line (i.e. cat ...) and the last line (i.e. EOF).

This technique can be used to create any text file you like.

cat << EOF > index.html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8">
    <title>Hello Site</title>
  </head>
  <body>
    <h1>Hello, Git!</h1>
    <p>This is my first version.</p>
  </body>
</html>
EOF

This technique is usefol for creating new files but is not useful when you want to edit an existing file. To do that the simplest thing to do is to use a text eidtor and described above.

6.6 2c) Commit the index.html file to the git repository:

git add index.html
git commit -m "Add basic index.html"

6.7 3) Add CSS

FULL FILE CONTENT BELOW — style.css

body { font-family: Arial, sans-serif; }
h1   { text-align: center; }
p    { color: #333; }

Link it in index.html (inside <head>):

<link rel="stylesheet" href="style.css">

Commit:

git add style.css index.html
git commit -m "Add basic styles and link CSS"

6.8 4) Add JavaScript

FULL FILE CONTENT BELOW — app.js

document.addEventListener('DOMContentLoaded', () => {
  console.log('Site loaded');
  const p = document.querySelector('p');
  p.textContent += ' (Now with JavaScript!)';
});

Link it in index.html before </body>:

<script src="app.js"></script>

Commit:

git add app.js index.html
git commit -m "Add simple JS to modify page text"

6.9 5) Review with status & log

  • Check changes:

    git status

  • See history:

    git log --oneline --graph --decorate --all

6.10 6) Publish to GitHub

6.10.1 Step 1 — Create an empty GitHub repo

  • On GitHub: New → Repository
  • Name it hello-site
  • Leave empty (no README/license)
  • Copy the repo URL

Choose HTTPS (PAT) or SSH.

6.10.2 Option A — HTTPS + Personal Access Token (PAT)

  1. Generate PAT:

    • GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
    • New token with repo scope

  2. Add remote & push:

    git remote add origin https://github.com/yourname/hello-site.git
git push -u origin main
    • Username → GitHub username
    • Password → paste PAT

  3. Later:

    git push

6.10.3 Option B — SSH Keys

  1. Generate:

    ssh-keygen -t ed25519 -C "you@example.com"

  2. Add to agent:

    eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

  3. Copy to GitHub:

    cat ~/.ssh/id_ed25519.pub

    Add via Settings → SSH and GPG keys.

  4. Test:

    ssh -T git@github.com

  5. Add remote & push:

    git remote add origin git@github.com:yourname/hello-site.git
git push -u origin main

6.10.4 Which method?

  • PAT (HTTPS): simplest start
  • SSH: convenient long-term

6.11 7) Daily workflow recap

  • Stage & commit:

    git add <file>
git commit -m "Message"

  • Inspect:

    git status
git log --oneline

  • Publish:

    git push

Commit it:

git add .gitignore
git commit -m "Add .gitignore for common clutter"

6.13 9) Cloning & pulling

6.13.1 Clone a repo

Instead of starting from scratch, you can copy a repo:

git clone https://github.com/yourname/hello-site.git
# or with SSH
git clone git@github.com:yourname/hello-site.git

This creates a new folder with all files and history.

6.13.2 Make changes & push

  • Enter the folder:

    cd hello-site

  • Change a file, commit, and push as before:

    git add index.html
git commit -m "Edit homepage"
git push

6.13.3 Pull changes from GitHub

If changes were made on GitHub (or another machine), bring them in:

git pull
