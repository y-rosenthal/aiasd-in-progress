10  10. Tutorial - intro

Material for this section is based on the following video:

  • YouTube - Supabase Full Product Tutorial (Jan 27, 2025)

The video gives a tutorial of how to build a full web app that has all of the following features and technology

  • website “frontend” built using NodeJS, npm, NextJS
  • storing data in a database (using Supabase’s “PostgreSQL” database hosting)
  • storing files on the server (using Supabase’s “buckets”)
  • users can login (i.e. authentication - using Google OAuth)
  • AI features (using an API connection to ChatGPT from OpenAI)
  • credit card charges (using Stripe payments)
  • edge functions to make the website faster and more secure (using Supabase’s edge functions)
  • automated “integration” tests - to make sure that everything works and that
    you don’t break the code when making changes

The video mentions a lot that you might not be familiar with. Don’t worry, this website will walk you through the harder parts and explain many concepts along the way that the video does not explain.

10.1 Code is provided - setup and integration is the hard part

The code is free for you to download and clone from GitHub here: https://github.com/pixegami/task-app (see the next section for how to do that).

The harder parts here are to understand

  • The architecture of the whole project and what different parts of the code are used for.

  • How to get this type of project running online. This includes creating accounts with Vercel (for web hosting), Supabase (for database server and other features - see above)

The goal is that once you understand the architecture of this project and setup the project online, you can use AI to create your own projects. You can use the AI to generate code, but it is important to udnerstand how the whole project is architected so that you know how to speak with the AI.

10.4 Overview

This tutorial will guide you through setting up a complete full-stack SaaS application using Next.js and Supabase. By the end, you’ll have a task management application with authentication, AI features, image uploads, and payment processing.

10.4.1 What You’ll Build

A “task management” application (i.e. a “todo app”) with the following features:

  • User authentication (including Google OAuth)
  • Task creation and editing
  • Automatic AI-powered task labeling
  • Image attachments for tasks
  • User profiles with task statistics
  • Monthly subscription plans with Stripe payments

The video assumes knowledge of things that you might not be familiar with. Don’t worry, this website will walk you through the harder parts and explain many concepts along the way that the video does not explain.
