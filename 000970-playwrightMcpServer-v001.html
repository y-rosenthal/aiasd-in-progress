11  9b. Playwright MCP server

The Playwright MCP server lets an MCP-capable client (e.g., Cursor) drive a real browser via Playwright tools. It exposes actions like navigate, click, type, evaluate JS, and more, using structured page snapshots (accessibility tree) rather than screenshots. :contentReferenceoaicite:0

11.1 Configure Cursor to use it

  1. Open Cursor ‚Üí Settings ‚Üí MCP ‚Üí Add new MCP Server. Choose command (STDIO). :contentReferenceoaicite:1

  2. Set:

    • Name: playwright
    • Command: npx
    • Args: @playwright/mcp@latest (You can also add it from the Playwright MCP README‚Äôs Cursor instructions.) :contentReferenceoaicite:2

Alternative (JSON style shown in some clients): json { "mcpServers": { "playwright": { "command": "npx", "args": ["@playwright/mcp@latest"] } } }

11.2 How to use it (in Cursor)

In a Composer chat, ask the agent to use playwright tools. Cursor will request approval before executing.

  • Example prompts:
    • ‚ÄúUse playwright to navigate to https://example.com.‚Äù
    • ‚ÄúClick the Sign in button, then type a test email.‚Äù
    • ‚ÄúRun browser_evaluate to read document.title.‚Äù
  • Common tools exposed by the server:
    • browser_navigate(url)
    • browser_click(element, ref)
    • browser_type(element, ref, text)
    • browser_hover(‚Ä¶)
    • browser_evaluate(function, [element|ref])
    • (and more, including drag, select option, screenshots)

Tip: If the tool list doesn‚Äôt show up, refresh the MCP servers list in Cursor and retry the prompt.

11.2.1 Example prompt you might use

You can try this prompt in Cursor:

Use the Playwright MCP server to navigate to https://example.com and log in with username ‚Äúuser‚Äù and password ‚Äúpass123‚Äù. Then take a screenshot of the webpage.

You may call the following tools:

  • browser_navigate(url)
  • browser_click(element, ref)
  • browser_type(element, ref, text)

Use them when appropriate.

The agent then uses the ‚Äúplaywright‚Äù server tools to carry out those steps. You may need to press ‚ÄúOK‚Äù (or similar) to allow the agent to carry out each of the steps. You can alternatively add the action to the ‚Äúwhitelist‚Äù.

11.2.2 Another example

Type something like:

‚ÄúOpen Wikipedia, search for Model Context Protocol,
click the first result, and tell me the title and first paragraph of the article.‚Äù

11.3 Tips

  • First try with simple pages (Wikipedia, example.com).
  • Keep one task per sentence.
  • Watch the tool calls Cursor proposes before approving them.

11.4 Important caveats & risks

Because this gives the model real browser control, you must thoroughly restrict allowed domains, audit tool calls, and use headless/isolated environments for safety. The MCP protocol has documented security risks.

11.5 Danger - enabling Playwright

Cursor can approve Playwright MCP tool calls automatically, but it‚Äôs not recommended outside of controlled demo or test setups.

You can disable the approval prompts in your MCP configuration, though doing so removes an important safety layer.

11.6 How to enable automatic approval

In your project‚Äôs .cursor/mcp.json (or global MCP settings file), find the Playwright entry and add "require_approval": "never" like this:

{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"],
      "require_approval": "never"
    }
  }
}

Then restart Cursor. Future Playwright tool calls will execute automatically without asking you each time.

11.6.1 Why it‚Äôs risky

A misinterpreted prompt could make the AI click or type on unintended elements.

Pages with ads, pop-ups, or login dialogs could trigger unexpected behavior.

Without manual review, it‚Äôs easier for the model to loop or send too many requests.

Safer compromise

If Cursor supports it, you can set:

"require_approval": "once"

That means approval happens only at the start of a session, not for every single tool call.

Best practice

Use ‚Äúnever‚Äù only with public, read-only sites (like Wikipedia or recipe pages).

Keep it on ‚Äúalways‚Äù for anything that could involve credentials, forms, or dynamic content.

Tips

Try grouping short actions into one natural-language request:

‚ÄúGo to AllRecipes, open the first pasta salad recipe, and summarize the ingredients.‚Äù

Cursor will often batch that into a single execution sequence, reducing approvals while keeping you safe.

11.7 Workaround idea

If you‚Äôre comfortable with advanced setup, you could define two MCP entries ‚Äî one ‚Äúsafe‚Äù and one ‚Äúinteractive‚Äù ‚Äî and use the one you need:

{
  "mcpServers": {
    "playwright-dangerous": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"],
      "require_approval": "never"
    },
    "playwright-safe": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"],
      "require_approval": "always"
    }
  }
}

Then choose which server to use in your prompt:

‚ÄúUse the playwright-safe server to open Wikipedia and summarize the first paragraph.‚Äù

Unfortunately, per-website approval rules are not supported.

Only per-server setting in mcp.json is supported.

Changing it via chat is not possible yet.
