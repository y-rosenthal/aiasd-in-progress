12  12. Step 0: Install Node.js and npm

In this project we will be using Node.js and the npm tool. Below we will explain what they are and how they are used. For now though, let’s first make sure that you have the necessary software installed.

  • First - understand that you only need to install Node.js. npm is automatically installed when you install Node.js.

  • To see if npm is already installed (and hence Node.js) type the following command at the Terminal prompt: npm -v

    • If npm is already installed you should see the npm version number. If you do, you can skip to the explanations of what Node.js and npm are.

    • If you didn’t see npm’s version number, then npm (and Node.js) is most probably not installed.

  • You can install Node.js from here - https://nodejs.org/en/download. Once you’ve installed Node.js you should have the npm tool also.

  • After you’ve installed nodejs shut your original Terminal and open a new Terminal window. Then run npm -v. You should see the version number for npm. If that worked great - otherwise try to figure out how to use npm with the help of your favorite AI (or knowledgeable human).

12.1 What is Node.js?

Node.js is a Javascript runtime environment. In other words, it is a program that runs Javascript code.

JavaScript was originally created to run inside web browsers to make webpages more interactive - e.g. dropdown menus, popup menus, tell users that they didn’t fill out a form properly, etc. It is still used inside browsers in these ways and others. To run Javascript in a browser, you do NOT need Node.js.

Node.js is used to run Javascript code outside of a browser. In this way you can use Javascript to create any type of program. This made Javascript more similar to other languages such as Python, Java, R, etc. Node.js was the first successful Javascript runtime. Today there are others such as Bun and Deno. However Node.js is still very popular.

12.2 What are Packages?

When developers build software, they often need to solve common problems: connecting to databases, handling dates and times, processing images, and so on. Rather than writing this code from scratch every time, developers create reusable bundles of code called packages (sometimes also called libraries).

If you’ve used R or Python, you’re already familiar with this concept: - In R, you might use packages like ggplot2 for visualization or dplyr for data manipulation - In Python, you might use pandas for data analysis or requests for making HTTP requests

JavaScript packages work the same way. For example, instead of writing complex code to handle dates, you could use a package like date-fns that someone else has already written and tested.

12.3 What is a Package Manager?

A package manager is a tool that helps you install, update, and manage packages. When you use R, you might run install.packages("ggplot2"). In Python, you use pip install pandas. These commands use package managers to:

  1. Download the package from a central repository
  2. Install it in the right location on your computer
  3. Keep track of which packages your project uses
  4. Manage dependencies (packages that other packages need to work)

12.4 What is npm (Node Package Manager)?

npm stands for “Node Package Manager”. It is package manager that comes with Node.js. When you install Node.js, npm comes with it automatically.

Later we will cover more about how to use npm. For now though we just want you to understand what it is. npm connects to a massive online repository containing hundreds of thousands of JavaScript packages. When you need a package, you simply run the following command from the CLI (e.g. Bash) to have npm download the package and make it available for your project to use.

npm install package-name

The npm registry is one of the largest package ecosystems in the world, with over a million packages available. This means that for almost any problem you’re trying to solve, someone has probably already created a package to help.
