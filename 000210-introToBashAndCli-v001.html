4  4. Intro to Bash (and CLIs)

The “user interface” on a computer is the mechanism with which users instruct the computer to do specific things. Non-technical users interact with computers by using the mouse to click and drag items on the screen. These users are using the “Graphical User Interface” (GUI) of the computer (see below for more info). Most people are not even aware that there is a way to do just about everything on a computer without even touching a mouse.

Technical users (e.g. programmers, computer system administrators, etc) often use a totally different mechanism to interact with the computer. They use a special program (known as a CLI or “command line interpreter”) to type special “commands” to do different things on the computer. For example, a technically inclined user, might type the command mkdir progressReports to create a new folder named progressReports. By contrast, non-technical users might right-click in the FileExplorer (Windows) or Finder (Mac) programs and choose “new file” from the menu that is displayed.

There many, many (hundreds?) of different commands that can be typed in the CLI program. Each command is used to accomplish something specific on the computer. Using a CLI is a skill, similar to programming, that takes some time to master. It is obviously much easier for a typical computer user to use the GUI on the computer. Most people aren’t even aware there is another way. Why then would technical users use a Comamnd Line Interface at all?

4.1 What are the benefits of using a CLI (Command Line Interpreter)?

It takes time and practice to master how to control a computer by using a CLI (command line interface). However, for technical users, the time spent learning how to use a CLI is time well spent. For the following reasons:

  • Many tools that are used in software development only have a CLI commands with no equivalent GUI program.

  • Once you have learned and mastered the numerous CLI commands, it can become trivial to accomplish very sophisticated results with much less effort than would be required if only using a GUI. A technical user might type the command mv project1/*2025* project2 to move all files with 2025 in their name from the project1 folder to the project2 folder. If there are 100 files in the project1 folder and 50 of those files have 2025 somewhere in their name, a typical non-technical user might take several minutes or longer to find each of the desired files and drag them to the correct folder. By contrast, typing the above command in the CLI and pressing ENTER takes a few seconds. This is a simple example. The general rule though is that for people who have mastered the CLI commands are able to control the computer in a much quicker and more efficient way.

  • It is relatively easy to “automate” the CLI commands. You can collect numerous CLI commands in a single file, known as a “script file”. Then you can direct the CLI to run all of the commands that are saved in the script file at once. This opens the door to writing script files that can be reused at different times or on different computers to accomplish a repetitive task.

4.3 What is a Graphical User Interface (GUI)?

The “user interface” on a computer is the mechanism with which the computer user instructs the computer to do specific things. Non-technical users interact with a computer via the “Graphical User Interface” of the computer. For example, all of the following are parts of the GUI (Graphical User Interface) on Mac and Windows

  • double clicking on icons to open a program or to open a file in the appropriate program
  • clicking/dragging on a titlebar to move a window to a different location
  • clicking on a filename in Windows Explorer or Mac Finder and dragging the file to different folder to either move or copy the file to the other folder
  • clicking on the “File” menu in Excel to get to the menu choices for opening/creating/etc files
  • clicking on different pictures and text to get something to happen
  • etc.

All of the above features rely heavily on using the mouse to interact with “pictures” or text that appear on the computer screen.

4.4 What is a CLI and Why Should You Care?

A CLI (Command Line Interface) is a way to interact with your computer using text commands instead of clicking on icons and buttons. Think of it like texting your computer instructions instead of pointing at things.

When people talk about “the command line” or “the terminal,” they’re usually referring to a CLI. It might look intimidating at first—just a black or white screen with text—but once you learn a few commands, it becomes a powerful way to work with files and folders on your computer.

4.4.1 What is Bash?

Bash (which stands for “Bourne Again Shell”) is one of the most popular CLIs. It’s the standard on most Linux systems and Mac computers. When you type commands in Bash, you’re essentially having a conversation with your computer in a special language it understands.

4.4.2 How to Access Bash on Different Systems

On Mac: Open the Terminal application. You can find it by pressing Cmd + Space and typing “Terminal”, or by going to Applications → Utilities → Terminal.

On Windows: Windows doesn’t come with Bash by default. You have a few options:

  1. Git Bash - A lightweight option that comes with Git. It’s perfect for basic Bash commands and is what we recommend for getting started.
  2. WSL2 (Windows Subsystem for Linux) - A full Linux environment inside Windows. This is more advanced and beyond the scope of this guide.

Important Note for Windows Users: Windows has its own native CLIs called CMD (Command Prompt) and PowerShell. These are conceptually similar to Bash—they let you type commands to control your computer—but the actual commands are different. For example, in Bash you use ls to list files, but in PowerShell you use Get-ChildItem or dir. This guide focuses on Bash commands.

4.6 Understanding Files and Folders

One important thing to understand: files can be arranged in different folders, which means two different files can have the same name as long as they’re in different folders.

For example, you could have: - projects/website/README.txt - projects/data-analysis/README.txt

These are two completely different files, even though they’re both named README.txt. They’re in different locations (different folders), so there’s no conflict.

4.7 Your Home Directory

Every user account on a computer has a special folder called the home directory. This is your personal space where you keep your files and folders.

  • On Mac/Linux: Your home directory is usually /Users/yourusername or /home/yourusername
  • On Windows (with Git Bash): It’s usually C:\Users\yourusername

When you open a new terminal, you usually start in your home directory automatically. Think of it as your “home base” in the file system.

4.8 The Bash Prompt

When you open your terminal, you’ll see something called the prompt. It might look like:

user@computer:~$

or

MacBook-Pro:~ username$

or just

$

Don’t panic if yours looks different! The prompt varies depending on your system settings, but it all works the same way. The prompt is just Bash’s way of saying “I’m ready for your next command!”

The $ symbol at the end is the most important part—it signals that you can type a command. When you see example commands in this guide or online, you’ll often see them written with a $ at the start. Don’t type the $—it’s just showing you where the prompt is.

4.9 Your First Commands: Where Am I?

4.9.1 The pwd Command (Print Working Directory)

The first thing you usually want to know is: where am I in the file system right now? That’s what pwd does—it shows you your current location.

$ pwd

Output:

/home/username/Desktop/bash-tutorial-files

This output is called a full path (or “absolute path”). Let’s break it down:

  • It starts with / which represents the very top (root) of your entire file system
  • Each / separates folder names
  • Reading left to right shows the path from the root to where you are now
  • This path says: “Start at the root, go into home, then username, then Desktop, then bash-tutorial-files”

Think of a full path like a complete street address: “123 Main Street, Springfield, IL, USA” gives you everything you need to find a location. Similarly, /home/username/Desktop/bash-tutorial-files gives you everything you need to find this folder.

4.10 Looking Around: The ls Command

Now that you know where you are, let’s see what’s in this location! The ls command lists the files and folders in your current directory.

$ ls

Output:

README.txt  documents  music  pictures  projects

Great! We can see there’s a README.txt file and four folders: documents, music, pictures, and projects.

4.10.1 Command Options: Making ls More Useful

Most Bash commands have options (also called “flags”) that modify their behavior. Options usually start with a dash -.

Let’s try ls with the -l option, which shows the “long” format with more details:

$ ls -l

Output:

total 4
-rw-r--r-- 1 username username    0 Oct 22 12:00 README.txt
drwxr-xr-x 4 username username 4096 Oct 22 12:00 documents
drwxr-xr-x 2 username username 4096 Oct 22 12:00 music
drwxr-xr-x 2 username username 4096 Oct 22 12:00 pictures
drwxr-xr-x 4 username username 4096 Oct 22 12:00 projects

Now we see much more information! The first column shows permissions, the numbers are file sizes in bytes, and we see dates and times. Lines starting with d are directories (folders), while lines starting with - are regular files.

Let’s add the -t option, which sorts by time (newest first):

$ ls -l -t

Output:

total 4
drwxr-xr-x 4 username username 4096 Oct 22 12:00 projects
drwxr-xr-x 2 username username 4096 Oct 22 12:00 pictures
drwxr-xr-x 2 username username 4096 Oct 22 12:00 music
drwxr-xr-x 4 username username 4096 Oct 22 12:00 documents
-rw-r--r-- 1 username username    0 Oct 22 12:00 README.txt

Now add -r to reverse the sort order (oldest first):

$ ls -l -t -r

Output:

total 4
-rw-r--r-- 1 username username    0 Oct 22 12:00 README.txt
drwxr-xr-x 4 username username 4096 Oct 22 12:00 documents
drwxr-xr-x 2 username username 4096 Oct 22 12:00 music
drwxr-xr-x 2 username username 4096 Oct 22 12:00 pictures
drwxr-xr-x 4 username username 4096 Oct 22 12:00 projects

4.10.2 Options Can Be Combined and Reordered!

Here’s something cool: the order of options doesn’t matter. These all do the same thing:

$ ls -l -t -r
$ ls -r -t -l
$ ls -t -r -l

Even better, you can combine single-letter options into one:

$ ls -ltr

This is exactly the same as ls -l -t -r! It’s just shorter to type. So -ltr means “long format, sorted by time, reversed order.”

4.11 Creating New Folders: The mkdir Command

The mkdir command lets you make a directory (create a new folder). Let’s create a new folder called temp:

$ mkdir temp

Output:

(no output - success is silent!)

In Bash, many commands don’t print anything when they succeed. No news is good news! Let’s verify it worked:

$ ls

Output:

README.txt  documents  music  pictures  projects  temp

There’s our new temp folder!

4.12 Moving Around: The cd Command

The cd command lets you change directory—that is, move to a different folder. Let’s go into the documents folder:

$ cd documents

Output:

(no output)

Again, no output means success. Let’s verify where we are now:

$ pwd

Output:

/home/username/Desktop/bash-tutorial-files/documents

Perfect! We’re now inside the documents folder. Let’s see what’s in here:

$ ls

Output:

notes  reports

We can see the notes and reports folders.

4.12.1 Using Full Paths with cd

You can jump directly to any location using a full path. Let’s go to the website folder:

$ cd /home/username/Desktop/bash-tutorial-files/projects/website
$ pwd

Output:

/home/username/Desktop/bash-tutorial-files/projects/website

4.12.2 Going Up One Level: The .. Shortcut

What if we want to go back up to the parent folder? The parent folder is the one that contains your current folder. For example, if you’re in website, the parent is projects.

Bash has a special shortcut: .. (two periods) means “the parent directory.” Let’s try it:

$ cd ..
$ pwd

Output:

/home/username/Desktop/bash-tutorial-files/projects

We moved up one level from website to projects! Let’s go up again:

$ cd ..
$ pwd

Output:

/home/username/Desktop/bash-tutorial-files

Now we’re back at the top of our tutorial files. You can even chain .. together. To go up two levels at once:

$ cd documents/reports
$ pwd

Output:

/home/username/Desktop/bash-tutorial-files/documents/reports

Now let’s go up two levels:

$ cd ../..
$ pwd

Output:

/home/username/Desktop/bash-tutorial-files

The path ../.. means “go up one level, then go up another level.”

4.12.3 Going Home

There are quick shortcuts to jump to your home directory:

$ cd ~

or just

$ cd

Both take you straight to your home directory, no matter where you are!

4.13 Getting Help: How to Learn More About Commands

You don’t need to memorize everything! Bash has built-in help for most commands.

4.13.1 The --help Option

Many commands support a --help option that shows you how to use them:

$ ls --help

Output:

Usage: ls [OPTION]... [FILE]...
List information about the FILEs (the current directory by default).
Sort entries alphabetically if none of -cftuvSUX nor --sort is specified.

Mandatory arguments to long options are mandatory for short options too.
  -a, --all                  do not ignore entries starting with .
  -A, --almost-all           do not list implied . and ..
  -l                         use a long listing format
  -r, --reverse              reverse order while sorting
  -t                         sort by time, newest first
  -S                         sort by file size, largest first
  -h, --human-readable       with -l, print sizes in human readable format
  ...

4.13.2 Understanding Help Output

Let’s break down what this help text means:

  • Short options: Single letters with one dash, like -l, -t, -r
    • These can be combined: -ltr is the same as -l -t -r
  • Long options: Full words with two dashes, like --all, --help, --human-readable
    • These usually can’t be combined
    • They’re more readable but take longer to type

For example, these do the same thing:

$ ls -a
$ ls --all

4.13.3 Man Pages: The Full Manual

If a command doesn’t have --help, or if you want even more detailed information, you can look up man pages (manual pages).

In a full Bash environment (like on Mac or Linux), you can type:

$ man ls

This opens a detailed manual page for the ls command. You can navigate with arrow keys and press q to quit.

Note: Some lightweight Bash environments (like Git Bash on Windows) might not have man pages installed. If man doesn’t work, you can search online for “bash ls man page” or “man ls” and find the documentation online.

4.14 Working with Files and Folders

Now let’s learn how to create, copy, move, and delete files and folders.

4.14.1 Creating Empty Files: The touch Command

The touch command creates an empty file or updates the timestamp of an existing file:

$ touch test.txt
$ ls

Output:

README.txt  documents  music  pictures  projects  temp  test.txt

We created a new empty file called test.txt!

4.14.2 Copying Files: The cp Command

The cp command copies files or folders. The syntax is: cp source destination

Let’s copy our test file:

$ cp test.txt test-copy.txt
$ ls

Output:

README.txt  documents  music  pictures  projects  temp  test-copy.txt  test.txt

Now we have both the original test.txt and a copy called test-copy.txt.

To copy a file into a folder:

$ cp test.txt documents/
$ ls documents/

Output:

notes  reports  test.txt

The test.txt file has been copied into the documents folder!

To copy an entire folder (directory) and all its contents, use the -r option (which stands for “recursive”):

$ cp -r documents documents-backup
$ ls

Output:

README.txt  documents  documents-backup  music  pictures  projects  temp  test-copy.txt  test.txt

4.14.3 Moving and Renaming: The mv Command

The mv command moves files or folders to a new location. It’s also used to rename things!

To rename a file:

$ mv test-copy.txt renamed-file.txt
$ ls

Output:

README.txt  documents  documents-backup  music  pictures  projects  renamed-file.txt  temp  test.txt

The file test-copy.txt has been renamed to renamed-file.txt.

To move a file into a folder:

$ mv renamed-file.txt temp/
$ ls temp/

Output:

renamed-file.txt

The file has moved into the temp folder. Notice it’s no longer in our current directory.

You can move and rename at the same time:

$ mv test.txt documents/my-test-file.txt
$ ls documents/

Output:

my-test-file.txt  notes  reports  test.txt

4.14.4 Removing Files: The rm Command

The rm command removes (deletes) files. Be careful! There’s no “undo” or trash bin—deleted files are gone forever!

$ rm documents/my-test-file.txt
$ ls documents/

Output:

notes  reports  test.txt

To remove multiple files at once:

$ cd temp/
$ touch file1.txt file2.txt file3.txt
$ ls

Output:

file1.txt  file2.txt  file3.txt  renamed-file.txt
$ rm file1.txt file2.txt file3.txt
$ ls

Output:

renamed-file.txt

To remove a folder and everything inside it, use -r (recursive) and usually -f (force, to avoid confirmation prompts):

$ cd ..
$ rm -rf documents-backup
$ ls

Output:

README.txt  documents  music  pictures  projects  temp

Warning: rm -rf is very powerful and very dangerous! Always double-check what you’re deleting. There’s a famous joke: “To understand what rm -rf / does, try it!” Don’t actually try it—it would attempt to delete everything on your computer!

4.14.5 Removing Empty Folders: The rmdir Command

The rmdir command removes directories, but only if they’re empty:

$ cd temp/
$ rm renamed-file.txt
$ cd ..
$ rmdir temp
$ ls

Output:

README.txt  documents  music  pictures  projects

If you try to use rmdir on a folder that has files in it, you’ll get an error:

$ rmdir documents

Output:

rmdir: failed to remove 'documents': Directory not empty

For non-empty folders, use rm -rf instead.

4.15 Reading Files: cat, less, and more

4.15.1 The cat Command

The cat command displays the contents of a file. It’s short for “concatenate,” but most people use it to quickly view file contents:

$ cd documents/notes/
$ cat ideas.txt

Output:

(displays the contents of ideas.txt)

If the file is empty (like ours), you won’t see anything! Let’s add some content first using a different method:

$ echo "My brilliant idea: automated coffee maker" > ideas.txt
$ cat ideas.txt

Output:

My brilliant idea: automated coffee maker

You can also use cat to view multiple files:

$ echo "More ideas here!" > ideas2.txt
$ cat ideas.txt ideas2.txt

Output:

My brilliant idea: automated coffee maker
More ideas here!

Limitation of cat: If the file is very long, cat dumps everything to the screen at once, and you’ll need to scroll back up to see the beginning. For long files, use less or more.

4.15.2 The less Command

The less command shows file contents one page at a time, letting you scroll through:

$ less ideas.txt

When using less: - Arrow keys or spacebar to scroll down - b to scroll back up - / followed by text to search - q to quit

less is great for long files because you have full control over navigation.

4.15.3 The more Command

The more command is similar to less but older and simpler:

$ more ideas.txt

With more: - Spacebar to go to the next page - Enter to go down one line - q to quit

Difference between less and more: - less lets you scroll backward; more only goes forward - less has more features (searching, better navigation) - The joke: “less is more, and more is less!” (meaning less is actually more capable than more)

In practice, most people use less because it’s more flexible. But both do the job for viewing files!

4.16 Quick Reference: Summary of Commands

Here’s a handy table of all the commands we covered:

Command What It Does Example
pwd Print working directory (show where you are) pwd
ls List files and folders ls -ltr
cd Change directory (move to another folder) cd documents
cd .. Go up one directory level cd ..
cd ~ or cd Go to home directory cd
mkdir Make a new directory mkdir new-folder
rmdir Remove an empty directory rmdir old-folder
touch Create an empty file touch file.txt
cp Copy files or folders cp file.txt copy.txt
cp -r Copy a folder and all contents cp -r folder/ backup/
mv Move or rename files/folders mv old.txt new.txt
rm Remove files rm file.txt
rm -rf Remove folders and contents rm -rf folder/
cat Display file contents cat file.txt
less View file with scrolling less file.txt
more View file page by page more file.txt

4.17 Tips for Learning

  1. Practice, practice, practice! The best way to learn Bash is to use it. Try the commands in the sample directory we provided.

  2. Don’t be afraid to experiment in test folders. Just be careful with rm commands!

  3. Use Tab completion: Start typing a file or folder name and press Tab. Bash will complete it for you! This saves time and prevents typos.

  4. Use the up arrow to cycle through previous commands. This is super handy when you need to run the same command again.

  5. Google is your friend: When you’re stuck, search for “bash how to [what you want to do]” and you’ll find lots of help.
