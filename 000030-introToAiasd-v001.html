2  2. Intro to Coding with an LLM

LLMs can act as pair programmers-explaining concepts, debugging, and generating code snippets.
Developers should verify AI-generated code and understand its logic, using LLMs as accelerators, not replacements.

2.1 What Is AI-Assisted Development?

  • Using AI tools (e.g., GitHub Copilot, ChatGPT, Tabnine) to help write, debug, and document code.

  • Works through natural language prompts → AI generates suggestions.

  • Goal: Accelerate productivity, not replace developers.

2.1.1 Benefits

  • Faster prototyping: Generate boilerplate & starter code.

  • Improved productivity: Offload repetitive coding tasks.

  • Learning aid: Explore unfamiliar libraries, frameworks, or languages.

  • Debugging help: Suggest fixes, highlight possible errors.

  • Documentation: Generate comments, READMEs, or summaries.

2.1.2 Limitations

  • Not always correct: AI can produce plausible but wrong code.

  • Security risks: May introduce vulnerabilities.

  • Bias & licensing: Trained on public data → risk of copyright/ethical issues.

  • Context gaps: AI may miss project-specific conventions or requirements.

2.2 Essential Technologies for AI-Assisted Development

2.2.1 Markdown

  • Lightweight markup language for formatting text
  • Standard for documentation, README files, and AI-generated content
  • Easy to read in plain text and renders beautifully when formatted

2.2.2 Command Line Interface (CLI) / Bash

  • Essential for efficient development workflows
  • Many of the more advanced AI coding tools integrate with terminal/command line
  • Bash scripting automates repetitive development tasks
  • Git version control operates primarily through CLI

2.2.3 Additional Development Tools

  • Version Control: Git and GitHub for code management
  • Package Managers: npm (Node.js), pip (Python), cargo (Rust)
  • Containerization: Docker for consistent development environments
  • Cloud Platforms: AWS, Google Cloud, Azure for deployment

2.3 AI-Assisted Development Workflow

2.3.1 1. Planning & Architecture

  • Use AI to brainstorm project structure
  • Generate technical specifications and documentation
  • Create user stories and feature requirements

2.3.2 2. Code Generation

  • AI can write functions, classes, and entire modules
  • Generate boilerplate code and configuration files
  • Create test cases and documentation

2.3.3 3. Debugging & Optimization

  • AI analyzes error messages and suggests fixes
  • Code review and improvement recommendations
  • Performance optimization suggestions

2.3.4 4. Learning & Documentation

  • AI explains complex code and concepts
  • Generates comprehensive documentation
  • Creates tutorials and learning materials

2.4 Best Practices for AI-Assisted Development

2.4.1 Effective Prompting

  • Be specific: Provide context, requirements, and constraints
  • Iterate: Refine prompts based on AI responses
  • Verify: Always review and test AI-generated code

2.4.2 Maintain Human Oversight

  • Understand the code before implementing
  • Test thoroughly: AI can introduce subtle bugs
  • Security review: AI may suggest insecure practices

2.4.3 Continuous Learning

  • Stay updated: AI tools evolve rapidly
  • Experiment: Try different AI models and approaches
  • Share knowledge: Collaborate and learn from others’ experiences

2.5 Other material

2.5.1 Best Practices

  • Always review AI-generated code.

  • Use AI for prototyping, scaffolding and exploration, not final production code.

  • Keep security in mind: sanitize input, check dependencies.

  • Iterate with prompts: refine instructions for better results.

  • Learn from it: treat AI as a teaching partner, not just a shortcut.

2.5.2 Buzzwords for best practices

  • Context Engineering

  • Spec Driven Development

2.5.3 Typical Workflows

  • Code completion: AI predicts the next lines.

  • Code generation: Write natural language → AI writes code.

  • Refactoring: Ask AI to improve readability or performance.

  • Explaining code: Have AI summarize or translate complex code.

  • Testing: Generate test cases & unit tests automatically.

2.5.4 The Human in the Loop

  • You remain responsible for correctness, quality, and ethics.

  • AI is a tool, not a replacement.

  • Think of AI like a junior developer: fast, but needs supervision.

2.5.5 Spec driven development

  • Github specify product

  • BMAD Method

    • follows Agile Engineering methods

    • praised by many in the space

  • numerous others

2.5.6 Key Takeaways

  • AI can speed up development, but requires critical oversight.

  • Use it to amplify creativity and productivity, not as a crutch.

  • The best developers will know when to trust AI—and when not to.
